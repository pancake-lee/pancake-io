## AI

- LLM (Large Language Model)
  - 通俗：CPU芯片
  - 大语言模型，基于 Transformer 架构的 AI 模型，是现代 AI 应用的“大脑”和基础，负责理解和生成文本。
  - OpenAI
    - 领先的 AI 研究机构和模型提供商，开发了 GPT 系列模型（可能是您提到的 "openclaw" 的误拼，或者是类似的 CrewAI 等框架）。
- Ollama
  - 通俗：大模型的“游戏模拟器” / 本地私有云 / 显卡上的 Docker
  - 一个开源的本地大模型运行框架，像安装普通软件一样简单地在个人电脑上部署和运行 Llama 3、Mistral 等开源大模型。
  - 定位：不需要联网、不需要付费 API，利用本地显卡/CPU 算力运行 AI，保护隐私且响应极快，并通过标准 API 供上层应用（如 LangChain、Dify）调用。
- LangChain
  - 通俗：主板 + 驱动 + 总线系统 = 让CPU能调用其他硬件
  - 一个流行的 AI 应用开发框架，提供了链（Chains）、记忆（Memory）、工具（Tools）等组件的抽象，用于简化 LLM 应用的构建。
  - RAG（检索增强生成）：LangChain 里的核心功能之一，解决 LLM「知识过时、不会查数据」的问题（比如让 LLM 先查你的公司文档，再回答问题）；
  - Tool Calling（工具调用）：Agent 的核心能力，由 LangChain 封装，让 LLM 能调用 API、数据库等外部工具；
- Agent (智能体)
  - 通俗：安装好系统的主机+智能助手
  - 拥有自主性的 AI 系统，以 LLM 为核心进行推理和规划，并能主动调用工具（Tools）与外部环境交互以完成目标。

### AI应用场景分类（按开发能力）

#### 非开发者：实时执行型 Agent

- 成本模型：每次执行任务都消耗 token，像雇佣助手一样持续付费。
- OpenClaw / AutoGPT
  - 通俗：随叫随到的 AI 助手，你说话它干活
  - 具体的 Agent 应用，每次任务都由 AI 实时思考、调用工具并执行，适合非技术人员的一次性任务（如数据收集、文档整理、简单自动化）。
  - 核心劣势：重复任务会重复烧 token，无法积累可复用的资产。
  - Tailscale：内网穿透工具，可将 OpenClaw 部署为远程服务。
- 适用场景：偶发性任务、探索性工作、没有编程能力的业务人员。

#### 有开发能力：代码生成型 AI 编程助手

- 成本模型：一次生成代码/脚本，后续可无限次免费执行，真正的投资而非消耗。
- GitHub Copilot
  - IDE 内的实时代码补全和生成工具，集成在 VS Code、JetBrains 等编辑器中，边写边提示。
  - 优势：基于当前代码上下文提供精准建议，适合快速编码和学习 API 用法。
  - 适用场景：日常开发、模板代码生成、API 探索。
- Cursor / Claude in VS Code
  - 通俗：能看懂整个项目的 AI 编程伙伴
  - 新一代 AI 编辑器（Cursor）或插件（Claude），支持多文件理解和大规模重构，可基于自然语言描述生成完整功能。
  - 优势：理解项目结构，生成可维护的工程级代码，比 Copilot 更适合复杂任务。
  - 适用场景：新功能开发、代码重构、技术栈迁移。
- Claude Code / ChatGPT
  - 独立的对话式 AI，可生成脚本、配置文件或完整应用模板。
  - 优势：无需 IDE，灵活性高，适合快速原型验证或一次性脚本。
  - 适用场景：Shell 脚本、SQL 查询、数据处理脚本、配置文件生成。

#### 中间地带：低代码/无代码平台

- n8n
  - 流水线搭建工具，可视化编排 API 调用、数据处理、定时任务，支持少量代码扩展。
  - 适用人群：懂基础逻辑但不擅长写完整程序的人。
  - 适用场景：后台自动化任务（如定时抓取数据、webhook 处理、多系统对接）。
- dify
  - AI 应用搭建平台，可快速构建聊天机器人、知识库问答等应用，支持 RAG 和工具调用配置。
  - 适用人群：产品经理、业务人员，想快速验证 AI 应用思路。
  - 适用场景：客服机器人、内部知识库、简单的对话式应用。

#### 核心观点总结

- 非开发者：选择 OpenClaw/AutoGPT，为即时结果付费，适合一次性任务。
- 有开发能力：选择 Copilot/Cursor/Claude，让 AI 生成可复用的代码资产，长期收益远超成本。
- 中间地带：选择 n8n/dify，用低代码平台快速搭建，兼顾效率和可维护性。

### Internet

- URL (Uniform Resource Locator)
  - 统一资源定位符，包含协议 (HTTP/HTTPS)、域名、路径等信息，用于标识互联网上的资源位置。
- DNS (Domain Name System)
  - 域名系统，负责将人类可读的域名解析为机器可识别的 IP 地址。
  - 解析过程涉及浏览器缓存、本地 HOSTS、本地 DNS 服务器以及根/顶级/权威域名服务器的查询。
- TCP 连接建立 (三次握手)
  - 建立可靠的连接，确保数据传输畅通。
  - 客户端与服务端通过发送 SYN 和 ACK 包进行三次交互。
- IP & 路由
  - 数据包封装 IP 头部（源/目的 IP），通过各级路由器在网络中转发，最终到达目的地。
- 后端处理
  - 服务端接收请求后，经过负载均衡、反向代理等架构层，最终由应用程序处理业务逻辑并返回 HTTP 响应。
- TCP 连接关闭 (四次挥手)
  - 优雅地关闭连接，确保双方都完成数据发送并确认。
  - 通信双方通过发送 FIN 和 ACK 包进行四次交互。
