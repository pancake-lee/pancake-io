在很长一段时间里，我做决策的方式可以概括为三个字：**凭直觉**。

看到一件事 → 脑子里快速过几个模糊的维度 → 形成一个综合感觉 → 拍板。效果不算差，毕竟直觉背后是经验的积累。但问题也很明显：**没法复盘，没法传递，也没法知道自己什么时候在自欺欺人。**

这篇文章记录了我从一个简陋的"三分法"出发，逐步演化出一套轻量级个人决策框架的过程。

## 一、三分法：直觉的显性化

最开始，我把所有决策的直觉结果归纳为三类：

```
积极（执行）
   △
  / \
 /   \
中立（待定）——— 消极（搁置）
```

每次面对一个决定，我会快速扫过几个隐性维度：

| 维度   | 内心问题           |
| ------ | ------------------ |
| 价值   | 做了有什么好处？   |
| 成本   | 时间/金钱/精力？   |
| 复杂度 | 好实现吗？         |
| 维护债 | 后面会不会很麻烦？ |
| 未知域 | 有没见过的坑吗？   |

然后综合出一个感觉，积极、中立、还是消极。

这个方法用了挺久，但它有三个致命缺陷：

1. **维度不固定**：这次想了成本，下次可能忘了，两次决策之间没有可比性
2. **权重不透明**：成本和价值到底哪个更重要？全凭当下的情绪
3. **无法复盘**：三个月后回头看，"我当时为什么选消极来着？"，忘光了
4. **无法传递**：别人问"为什么不做"，只能说"直觉"，听起来像借口

我意识到需要一套**足够轻量、但能显性化思考过程**的方法。太重的框架（比如 AHP 层次分析法）对日常决策是过度设计，我不可能为了决定"要不要重构这个模块"花半小时打分。

## 二、为什么是 ICE

对比了几种常见的决策框架：

| 方法            | 复杂度 | 与三分法的契合度 | 选择理由                                       |
| --------------- | ------ | ---------------- | ---------------------------------------------- |
| RICE            | 中     | 高               | 多了 Reach（触达人数），个人场景价值维度已覆盖 |
| ICE             | 低     | **最高**         | 三个维度直接对应三分法的核心考量               |
| 加权决策矩阵    | 中     | 中               | 需定义权重，个人场景过度设计                   |
| 成本-价值四象限 | 低     | 中               | 只有两维，丢失 Confidence（未知域）            |

**ICE 是三分法的最小系统化升级**，它比四象限多了一个关键维度（把握度），又比 RICE 少了个人场景不需要的 Reach。

### ICE 三维度

| 维度           | 含义      | 问题             | 1-5 评分标准                     |
| -------------- | --------- | ---------------- | -------------------------------- |
| **I**mpact     | 影响/价值 | 做了有多大好处？ | 1=几乎无感，5=核心目标直接推进   |
| **C**onfidence | 把握度    | 有多大把握做成？ | 1=完全未知，5=做过类似，路径清晰 |
| **E**ffort     | 成本      | 要投入多少？     | 1=几小时搞定，5=数周以上         |

### 评分到行动的映射

| ICE 组合           | 解读                   | 行动                    |
| ------------------ | ---------------------- | ----------------------- |
| I≥4, C≥4, E≤2      | 高价值、高把握、低成本 | **立即执行**            |
| I≥3, C≥3, E≤3      | 价值可接受，有一定把握 | **规划后执行**          |
| I≥3, 但 C≤2 或 E≥4 | 有价值，但未知或太重   | **先研究/降低不确定性** |
| I≤2, 或 E≥4 且 C≤2 | 价值低或风险过高       | **搁置/放弃**           |

和三分法的对应关系很自然：

```
原三分法          ICE 评分              行动
─────────────────────────────────────────────────
积极（执行）  ←→  I高 C高 E低      →  立即执行
              ←→  I中 C中 E中      →  规划后执行

中立（待定）  ←→  I中 C低/E高      →  先研究

消极（搁置）  ←→  I低 或 E高+C低   →  明确放弃
```

## 三、关键概念：关键假设 vs 反向论点

框架搭起来之后，我发现光有评分还不够。两个容易混淆但作用不同的概念，是让这个框架从"打分游戏"变成"真正有用的决策工具"的关键。

| 概念         | 问题                                 | 作用                             | 例子                                     |
| ------------ | ------------------------------------ | -------------------------------- | ---------------------------------------- |
| **关键假设** | 我 Confidence 低是因为我不知道什么？ | 揭示不确定性的来源，用于后续验证 | "这个库是否兼容我们现有的认证中间件？"   |
| **反向论点** | 这件事可能因为什么失败？             | 强迫自己找反对理由，避免乐观偏差 | "这个库可能维护不活跃，半年后成为技术债" |

> **记忆技巧**：关键假设 = 不知道的**事实**；反向论点 = 可能发生的**坏结果**。

这两个概念来自一次与亚马逊 PR/FAQ 决策框架的对比讨论。PR/FAQ 的核心理念是"先写新闻稿，再反向推导风险"，但那是团队决策的尺度。对个人来说，关键假设和反向论点已经足够覆盖"不确定性管理"和"乐观偏差纠正"两个核心需求。

另外还有一个重要的区分：**双向门 vs 单向门**。低试错成本的决策（比如换一个终端工具、试一个开源库）属于"双向门"，进去了可以出来，不要过度研究，直接试错即可。ICE 评分更适合"单向门"，一旦开始就不好回头的那种。

## 四、使用流程

### 快速版（日常决策，≤2 分钟）

1. **一句话**：我要决定的是什么事？
2. **三个数字**：I=? C=? E=?（1-5，凭直觉）
3. **查表**：对应什么行动？
4. **关键假设**：如果 C ≤ 3，写下"我不确定的是：\_\_\_"
5. **反向论点**：写 1-2 条"不做这件事的理由"
6. **记录**：丢进 todo 或笔记

### 完整版（重要决策，≤10 分钟）

1. **定义选项**：只有一个？还是多个方案对比？
2. **每个选项打 ICE 分**
3. **计算 ICE 值**：`I × C / E`（可选，用于多选项排序）
4. **写下关键假设**：每个 C 低的原因，以及如何验证
5. **写下反向论点**：至少 2 条，其中一条是"最坏情况"
6. **决策 + 记录 + 设复盘日期**

## 五、记录模板

以下是日常用的模板，直接粘贴到笔记里即可：

```
【决策】：
【I/C/E】：  /  /
【行动】：执行 / 待研究 / 搁置
【关键假设】：（如果 C≤3）我不确定：____
【反向论点】：
  - 不做理由1：
  - 不做理由2：
【复盘日期】：
```

实际使用中，最快的记录可以只有一行：`[E-2026-06-27] 升级 Go 版本到 1.24 — I3/C4/E1 — 执行`。关键不是格式，是**写下来这个动作本身**，它逼你把模糊的直觉转化成明确的判断。

## 六、边界与注意事项

1. **ICE 不是万能**：情感决策（人际关系）、价值观冲突（职业选择）不适合纯量化
2. **Confidence 最难打**：承认"我不知道"是高价值行为，如果明明不确定却打高分，ICE 会骗你
3. **Effort 要诚实**：包括后续维护成本，不只是 upfront 投入
4. **记录比评分重要**：打分过程逼你显性化思考，分数本身只是副产品

---

这个框架从最初的三分法到现在的版本，经历了约两个月的迭代。最大的收获不是"决策更准了"，那很难衡量，而是**每次决策都有了可回溯的轨迹**。三个月后翻开笔记，能看到当时的 reasoning，知道"原来我当时是这么想的，但后来发现假设 X 不成立"。

这种积累，才是决策能力真正提升的来源。
