刚接触结构化思维的人，很容易陷入一种焦虑：方法太多了。

金字塔原理、MECE、5W2H、六顶思考帽、第一性原理、二阶思维、逆向思维、帕累托法则、艾森豪威尔矩阵、RICE 评分、MoSCoW、决策矩阵……光记名字就够累的，更别说在合适的时候挑出最合适的那一个。

为了日常小问题更容易使用这些工具，我自己做了点整理，分为三个部分：一个完整的知识地图、一套默认流程、一张卡壳速查表。

## 一、知识地图：所有方法的归属结构

先把所有方法塞进一张图里，知道谁是谁、谁管谁。以后听到一个新名词，你能立刻判断它在这个体系里占哪个位置。

```
结构化思维
│
├── 一、核心：分类拆分（骨架系统）
│   └── 金字塔原理
│       ├── 纵向：结论先行，以上统下
│       │   └── 拆解工具：逻辑树 = 议题树 = 鱼骨图 = 5Why（同根不同形）
│       ├── 横向：归类分组，逻辑递进
│       │   ├── 分类标准：MECE（不重叠、不遗漏）
│       │   └── 发散工具：思维导图（创意阶段用，收敛后回金字塔）
│       ├── 序言结构：SCQA = PREP（完整版 vs 快节奏版）
│       └── 精简原则：奥卡姆剃刀（别过度拆解）
│
├── 二、辅助：思考深度（血肉填充）
│   ├── 穷尽维度：5W2H（7 个角度检查清单）
│   ├── 切换视角：六顶思考帽
│   ├── 回归本质：第一性原理
│   ├── 追问深度：二阶思维（"然后呢？"）
│   └── 反面验证：逆向思维 / 红队演练
│
└── 三、辅助：表达与决策（输出转化）
    ├── 优先级排序
    │   ├── 帕累托法则（抓关键 20%）
    │   ├── 艾森豪威尔矩阵（紧急 × 重要）
    │   ├── RICE 评分（Reach × Impact × Confidence / Effort）
    │   └── MoSCoW（Must / Should / Could / Won't）
    └── 决策评估
        ├── 决策矩阵（多因素 × 权重 × 打分）
        └── 利弊清单（两列对比）
```

这张图的核心逻辑，我理解是这样的：**金字塔原理更像是操作系统，其他工具则是应用程序**。日常运行系统，卡壳时启动应用。

需要提醒的是：这个体系里只有《金字塔原理》和《六顶思考帽》是完整专著，其余都是工具或原则，大部分一段话就能讲清，不需要专门买书来看。

## 二、默认流程：80% 的场景只需要三步

面对大部分问题，先跑下面这三步，不纠结、不选工具：

> 这里场景其实还要细分，但目前还没有理得这么深，场景可以再分为：
> 察觉 → 厘清 → 拆解 → 分析 → 决策 → 表达 → 执行 → 复盘
> 我们先不纠结哪个工具适用于哪个场景，给自己一点容错控件，
> 实践之后，让大脑自己找到好用的工具。

| 步骤                 | 动作                       | 核心问题                                       |
| -------------------- | -------------------------- | ---------------------------------------------- |
| Step 1：金字塔框定   | 结论先行，以上统下         | "我的核心观点是什么？有哪 3 个支撑理由？"      |
| Step 2：MECE 检查    | 横向**分组**不重叠、不遗漏 | "这 3 个理由有没有交集？有没有漏掉关键维度？"  |
| Step 3：逻辑树补深度 | 逐层拆解到可**行动**       | "哪个理由需要拆细？拆到谁、什么时候、做什么？" |

**我自己的体会是，80% 的日常场景，这三步就够了。** 目标是练到不用想，像开车踩离合一样自然。

如果你刚开始练，可以拿日常工作里的任何一件事来练：写周报前先定结论、开技术评审前列三个支撑论点、做方案时用 MECE 检查有没有漏维度。练五次之后，这三步会开始自动化。

## 三、卡壳速查表：精准调用，不乱翻箱

默认流程走不通时，再翻工具箱。关键不是"想到哪个用哪个"，而是**识别卡顿感，精准调用**：

| 卡顿感                   | 调用工具        | 怎么用                                             |
| ------------------------ | --------------- | -------------------------------------------------- |
| 结论站不住脚，像在抄答案 | 第一性原理      | "抛开所有类比，最基本的事实是什么？能推出什么？"   |
| 理由有遗漏，总觉得缺了啥 | 六顶思考帽·白帽 | "大家只摆事实，不推理，补全信息缺口"               |
| 团队吵成一团，各说各话   | 红队演练 / 黑帽 | "接下来 10 分钟只准挑刺，证明这个方案会死"         |
| 选项太多，不敢拍板       | 决策矩阵        | "列标准、赋权重、打分算总分，让数字说话"           |
| 时间紧迫，30 秒内要说服  | PREP            | "观点 → 理由 → 例子 → 重申，一句废话没有"          |
| 对方没兴趣，听不进去     | SCQA            | "先讲冲突/变化，再抛问题，最后给答案"              |
| 需求做不完，资源不够     | MoSCoW          | "先标 Must have，其余排队，明确说 No"              |
| 想突破创新，被现状困住   | 思维导图 / 绿帽 | "先发散 20 个疯狂想法，不评判，再收敛"             |
| 问题反复出现，治标不治本 | 5Why            | "连问五个为什么，追到根因再动手"                   |
| 事情太多，不知道先做哪个 | 艾森豪威尔矩阵  | "紧急重要立即做，重要不紧急计划做，其余委托或砍掉" |
| 想抓关键，避免平均用力   | 帕累托法则      | "找出带来 80% 结果的 20% 因素，all in"             |

这张表的价值不在于全面，而在于**把"感觉卡住了"翻译成"该用哪个工具"**。下次卡住时，先别想"我去翻翻有哪些方法"，而是先问自己"我的卡顿感属于哪一列"，然后精准调用那一行。
